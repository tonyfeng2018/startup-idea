# Startup Idea Diagnose Skill

## 概述

这是一个帮助早期创业团队系统性验证想法的AI Skill。

## 适用场景

- 创业者说"我有一个想法"
- 创业者说"帮我看看这个想法靠谱吗"
- 创业者说"早期团队如何验证想法"
- 涉及创业、想法、痛点、用户验证等话题

## 使用条件

### 最大化效果的条件

| 条件 | 说明 | 重要性 |
|------|------|--------|
| **模型推理模式** | 建议开启推理模式，能更好进行追问 | ⭐⭐⭐⭐⭐ |
| **上下文长度** | 建议32K以上，支持完整对话 | ⭐⭐⭐⭐ |
| **多轮对话** | 此Skill需要多轮交互，不适合单轮 | ⭐⭐⭐⭐⭐ |
| **工具支持** | 建议支持memory相关工具，用于记录 | ⭐⭐⭐ |

### 推荐模型

- Claude 系列（推理模式开启）
- GPT-4（带推理）
- 国产：豆包/GLM（带推理）

## 核心框架

### 阶段一：想法收集

首先了解项目阶段：

> 你的项目目前处于什么阶段？
> - A) 想法阶段 — 刚有想法，还没开始做
> - B) 原型阶段 — 有Demo/原型，在测试
> - C) 早期用户 — 有少量用户，在迭代
> - D) 已有收入 — 开始有付费用户

---

### 阶段二：六问诊断法

#### Q1: 真实痛点

**问：** "告诉我一个具体场景，你/别人遇到了什么问题？这个场景发生时，人在干嘛，什么感受？"

**追问技巧：**
- "这个人当时在用什么替代方案？"
- "这个问题多久出现一次？"
- "这个问题严重到什么程度？"

---

#### Q2: 目标用户

**问：** "谁会有这个痛点？能不能给我一个具体的人名和背景？"

**追问技巧：**
- "这个人做什么工作？"
- "这个问题和他的工作/生活什么关系？"
- "这个人会为解决这个问题花多少钱？"

---

#### Q3: 替代方案

**问：** "现在没有你的产品时，这个人怎么解决这个问题的？"

**追问技巧：**
- "这个替代方案有什么不好？"
- "为什么他们不用更好的方案？"
- "你有什么他们没有的？"

---

#### Q4: 最小可行

**问：** "如果只能做一个功能解决这个痛点，是什么？"

**追问技巧：**
- "这个功能做出来需要多久？"
- "用户需要做什么才能用上这个功能？"
- "没有这个功能会怎样？"

---

#### Q5: 验证证据

**问：** "到目前为止，你为这个想法做了哪些验证？"

---

#### Q6: 下一步

**问：** "如果现在要验证这个想法，最小的一步是什么？"

---

### 阶段三：假设清单

```
假设清单：
1. [假设：目标用户确实有这个痛点] — 需要验证
2. [假设：用户愿意为这个付费] — 需要验证
3. [假设：我们的解决方案比现有方案好] — 需要验证
4. [假设：我们能接触到这些用户] — 需要验证
```

---

### 阶段四：诊断报告模板

```markdown
# 想法诊断报告

## 想法概述
[一句话描述这个想法]

## 痛点评估
- 痛点真实性：⭐⭐⭐⭐⭐ (1-5)
- 痛点频率：高/中/低
- 痛点付费意愿：强/中/弱

## 用户评估
- 目标用户清晰度：⭐⭐⭐⭐⭐
- 用户可触达性：容易/一般/困难

## 竞争格局
- 主要竞争对手：[替代方案]
- 差异化：[你的优势]

## 关键假设
| 假设 | 验证状态 | 风险 |
|------|----------|------|
| 假设1 | 待验证 | 高/中/低 |
| 假设2 | 已验证 | - |

## 建议的下一步
1. [最小验证动作]
2. [次小验证动作]
3. [如果验证成功，下一步]

## 诊断结论
[总结：这个想法目前处于什么状态，需要优先验证什么]
```

## 使用示例

### 输入

> "我有一个想法，做一个面向中小企业的AI客服系统，你觉得靠谱吗？"

### 理想输出（AI引导）

> "好的，让我们来诊断一下这个想法。首先，**你的项目目前处于什么阶段？**
> - A) 想法阶段 — 刚有想法，还没开始做
> - B) 原型阶段 — 有Demo/原型，在测试
> - C) 早期用户 — 有少量用户，在迭代
> - D) 已有收入 — 开始有付费用户"

（根据用户回答，继续追问...）

---

## 注意事项

1. **一次只问一个问题** — 不要把所有问题一次抛出
2. **追问直到具体** — "用户"→"什么人"→"具体谁"
3. **不替用户做决定** — 只呈现分析框架，决策权在用户
4. **给出具体行动** — 每个诊断必须有下一步可执行的动作
