# Startup Idea Validator

AI创业想法验证智能体 — 七维评估体系，数据驱动，决策导向。

> **当前版本**：v5（2026-04-02）  
> **适用场景**：创业想法验证 / 痛点分析 / 竞品调研 / 市场规模评估

---

## 🎯 项目介绍

这是一个基于 AI 的创业想法验证 Skills，帮助创业者在**30分钟内**系统性验证一个想法是否靠谱。

核心方法论来自 YC + A16Z，结合中国市场实际情况，输出结构化的七维评估报告 + 可执行的最小验证动作。

---

## 📊 核心功能

| 功能 | 说明 |
|------|------|
| **七维评估体系** | 痛点 / 差异化 / 时机 / 团队 / 商业 / 竞争壁垒 / 风险合规 |
| **并行搜索6项强制规则** | 竞品 / 市场规模 / 渗透率 / 定价 / 差评 / 融资，必须全部完成再生成报告 |
| **竞品 Feature Matrix** | 6个竞品全覆盖（功能/防作弊/定价/多语言/集成/市场份额/差评痛点）|
| **SWOT + 关键假设清单** | 3-5条假设 + 验证方式 + 状态（✅已验证/❓待验证）|
| **搜索失败透明处理** | 缺失数据标注[待验证]，提供根本解决方案和具体验证平台 |
| **最小验证动作** | 带时间/平台/关键词的具体可执行任务 |

## 🚀 快速开始

### 使用示例

```
在 OpenClaw / Claude Code 中发送：

帮我看看这个想法靠不靠谱：做一个面向HR的AI面试工具
```

### 输出内容

1. **七维评分**：痛点/差异化/时机/团队/商业/竞争壁垒/风险合规
2. **竞品 Feature Matrix**：6个竞品对比
3. **SWOT 分析**：一句话总结
4. **关键假设清单**：3-5条待验证假设
5. **差异化方向**：2个推荐垂直赛道
6. **最小验证动作**：带时间/平台/关键词的具体任务
7. **完整报告**：自动存档到 `memory/` 目录

## 🔑 核心设计原则

1. **数据驱动** — 每个评分必须有量化依据，无法量化则标注[待验证]
2. **诚实比完整更重要** — 宁可承认数据缺失，也不要假装有数据
3. **搜索失败是常态** — 必须设计降级方案，不能依赖单一搜索工具
4. **验证方式比数据更重要** — 用户拿着"待验证"标注可以去自行核实
5. **模板是骨架，思考是灵魂** — 结构化输出是下限，深度分析才是价值

---

## ⚠️ 数据说明

本项目中的报告和模板默认依赖外部搜索工具获取实时数据。

当搜索工具无法访问时，所有量化数据会标注 `[待验证]`，并提供具体验证平台（官网/报告名称/搜索关键词），确保可追溯和可补充。
